================================================================
  ① search_log — 追查「庫存對不上」(RCA) ★最有戲
================================================================
玩法：問「為什麼少 / 對不上」，它會翻交易檔→比對採購單→找出哪批短收。

▼ 會真追到原因的商品（這些帳是故意設成對不上的）：
   智慧手環、抗菌洗衣精 4kg、蚊香液補充瓶、彈力健身環、嬰兒紙尿布 L

▼ 範例句（都會中）：
   智慧手環怎麼少這麼多
   抗菌洗衣精帳對不上
   蚊香液補充瓶是不是少貨了
   彈力健身環為什麼短少
   嬰兒紙尿布 L少貨原因
   查一下智慧手環為什麼短少
   幫我追抗菌洗衣精的扣帳異常
   智慧手環入庫數量不對
   蚊香液補充瓶這個月入庫對不上
   彈力健身環短收了嗎
   嬰兒紙尿布是誰動的
   追智慧手環的庫存差異
   抗菌洗衣精怎麼會對不起來

▼ 查正常商品（會回「沒發現異常」，證明它不亂報）：
   瑜珈墊這週有異常嗎
   藍牙耳機的進出記錄查一下


================================================================
  ② manage_config — 讀/改設定 ★會寫入、跳確認框
================================================================
玩法：改設定是危險動作，它會先給你看「要改幾項、改成什麼」，
      按【✅ 確認執行】才真的寫入，並自動 .bak 備份 + 記 audit log。

▼ 招牌題（一句改一整倉）：
   南倉安全庫存全部加30
   北倉安全庫存全部加20
   中倉的安全庫存都加50

▼ 改單一商品：
   把無線藍牙耳機安全庫存改成50
   無線藍牙耳機安全庫存設30
   把玻璃保鮮盒 5入安全庫存改成20
   南倉的強力垃圾袋 50入安全庫存改成80
   中倉的桌上型 USB 風扇安全庫存改成20

▼ 改營運參數：
   補貨前置天數改成7
   前置天數設10
   把前置天數調到5

▼ 查設定（不會跳確認、直接回答）：
   現在的補貨前置天數是多少
   前置天數現在設定
   目前安全水位倍數設定
   看一下補貨目標天數
   無線藍牙耳機的安全庫存是多少
   折疊露營椅的安全庫存是多少


================================================================
  ③ run_script — 跑白名單腳本 ★白名單沙盒
================================================================
玩法：只能跑「白名單裡的腳本」，名單外一律擋。會跳確認後執行。

▼ 白名單內（會執行）：
   幫我跑一次月底盤點
   執行月底盤點
   跑一下盤點作業
   匯出進出記錄
   把異動匯出來
   匯出這個月異動
   匯出庫存異動
   重新產生種子資料
   重新生成資料
   重跑種子資料
   export movements
   regenerate seed data

▼ 白名單外（會被擋，展示安全性）：
   幫我刪掉北倉所有庫存
   執行 rm -rf
   → 回：「不在可執行白名單內。可用：月底盤點 / 匯出進出記錄 / 重產種子資料」


================================================================
  ④ generate_report — 產生報告檔 ★Agent 自己寫檔
================================================================
玩法：掃全倉→彙整→寫成 markdown 報告檔到 reports/，畫面可預覽。

▼ 全倉體檢報告（4 段：總覽/缺貨/到期/PO異常）：
   幫我出個全倉體檢報告
   產生全倉報告
   倉庫狀況報告
   整理報告給我
   報表輸出一下
   我要一份報告
   彙整一份全倉報告
   輸出全倉報表
   做個盤點報告
   export inventory report

▼ 指定類型報告：
   出個缺貨報表          （只出缺貨）
   補貨清單報表          （只出缺貨）
   低庫存報表            （只出缺貨）
   到期商品報告          （只出到期）
   效期警示報表          （只出到期）
   異常對帳報告          （只出 PO 短收）
   對不上的彙整報告      （只出 PO 短收）


================================================================
  ⑤ list_files — 動態列出有哪些資料可查 ★Agent 自己看檔案系統
================================================================
玩法：讓它自己 ls 倉庫資料夾，看現在有哪些檔可讀（限沙盒內）。

▼ 看總覽（列出所有資料區）：
   倉庫裡有哪些紀錄檔
   有哪些資料可以看
   有什麼可以查的
   列出檔案
   有哪些目錄可以看
   資料夾裡有什麼
   看看有什麼資料

▼ 看特定區：
   transactions有哪些檔
   交易紀錄檔有哪些
   進出紀錄檔列一下
   訂單檔有哪些
   主檔有什麼
   有哪些報告
   看一下報告目錄
   異動紀錄檔列一下


================================================================
  ⑥ 主動異常偵測 — 不用問、系統自己掃 ★最像真監控系統
================================================================
玩法：① 一進站，頂部自動彈「⚠ 主動偵測到 N 項異常」橫幅（點可展開）
      ② server 背景每 5 分鐘自動掃一次，掃到新異常即時更新
      ③ 也可手動拉：瀏覽器開 http://127.0.0.1:8000/anomalies

▼ 偵測 5 類（目前資料：6 嚴重 + 7 警告 + 41 注意）：
   🔴 PO 短收/對不上     （採購單應收≠實收）
   🔴🟠🟡 快斷貨/低於安全線（撐天 ≤7紅 / ≤14橙 / 低於安全線黃）
   🟠 出庫暴增/暴跌      （偏離日均 >3σ，可能刷單/失竊）
   🟠 快到期囤貨         （效期≤14天還有量，恐報廢）
   🟡 呆滯品            （長期零出庫佔庫存）

▼ 外部推播（要接時，設環境變數即生效，不用改 code）：
   Slack/Teams：  設 SLACK_WEBHOOK_URL
   LINE：         設 LINE_NOTIFY_TOKEN
   Email：        設 SMTP_HOST / SMTP_USER / SMTP_PASS / ALERT_TO


================================================================
  v1 原有 7 大查詢（都還在、沒回歸）
================================================================
▼ query_inventory（查庫存）：
   無線藍牙耳機庫存 / 充電線現在幾件 / 北倉電子 / 南倉洗衣精多少 / 看一下牛仔褲庫存

▼ query_movement（查進出）：
   這週出倉記錄 / 本月出入總量 / 這個月運動進貨出貨 / 全自動咖啡機出了幾件

▼ list_low_stock（查缺貨）：
   南倉需要補貨的 / 北部倉缺貨清單 / 食品類有沒有快缺貨的 / 服裝缺貨清單

▼ compare_warehouses（比倉）：
   北倉和南倉比一下 / 南倉vs中區倉 貨值 / 中倉跟南部倉哪個數量低

▼ list_hot_items（熱賣/滯銷）：
   這週哪些賣最好 / 本月最熱門排行 / 本週哪些商品滯銷 / 這月滯銷排行

▼ query_related_items（連帶分析）：
   買藍牙耳機的人也買了什麼 / 慢跑鞋連帶分析 / 跟毛帽一起賣的有哪些

▼ list_expiring_items（到期）：
   快到期的有哪些 / 有什麼快過期 / 即期品清單 / 保存期限快到的
